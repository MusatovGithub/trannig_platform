# Временный пароль и обязательная смена

## Что реализовано

- При создании/переотправке доступа пользователю выдается временный пароль.
- В БД сохраняется флаг обязательной смены и срок действия временного пароля.
- В Redis сохраняется дополнительный маркер на 8 часов.
- После входа пользователь с активным флагом автоматически перенаправляется на обязательную смену пароля.
- После успешной смены пароля флаг/срок в БД и маркер в Redis удаляются.

## Где используется

- `authen/temp_password.py`:
  - `set_temporary_password_state(user)` — ставит флаг, срок и Redis-маркер.
  - `is_temporary_password_expired(user)` — проверяет срок действия.
  - `clear_temporary_password_state(user)` — очищает флаг, срок и Redis-маркер.
- `authen/views.py`:
  - В `sign_in` добавлена проверка маркера и редирект на `force_password_change`.
  - Добавлена view `force_password_change` для обязательной смены пароля.
  - После успешной смены пароля маркер очищается.
- `custumer/views.py`, `employe/views.py`:
  - После генерации временного пароля вызывается `mark_temporary_password`.
  - Обновлен текст письма и добавлены ссылки на вход/смену пароля.

## Поведение пользователя

1. Пользователь получает письмо с временным паролем.
2. Входит в систему.
3. Если маркер временного пароля активен, система переводит на страницу смены пароля.
4. После успешной смены пароля маркер удаляется, дальнейшая работа идет в обычном режиме.
5. Если 8 часов истекли до смены, вход с таким временным паролем блокируется, требуется переотправка доступа.

## Примечания по доставляемости

- Текст письма сделан нейтральным и информационным.
- Добавлены явные шаги для пользователя и причина получения письма.
- Письмо отправляется в формате text + html.
- Добавлены заголовки `Reply-To` и `List-Unsubscribe`.
- Для стабильной доставляемости обязательно держать в порядке SPF/DKIM/DMARC.
